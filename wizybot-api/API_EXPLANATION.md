# WizyBot API - Technical Documentation

This document explains every step and component of the Wizybot API in a simple, easy-to-understand way.

## 1. Entry Point & Setup (`src/main.ts`)
The `main.ts` file is where the application boots up. 
- **NestFactory.create**: Initializes the NestJS server.
- **CORS (`app.enableCors()`)**: Cross-Origin Resource Sharing. This allows web browsers (like a frontend app running on a different port or domain) to make requests to this API without being blocked by browser security policies.
- **Global Pipes (`app.useGlobalPipes`)**: Enables automatic validation across the whole app. It ensures that incoming data matches our rules before it hits the controller.
- **Swagger**: Automatically generates documentation and a testing UI (available at `/api/docs`) so developers can see what the endpoints look like and test them.

## 2. Global Configuration (`src/app.module.ts`)
The root module of the application.
- **ConfigModule**: Loads environment variables from the `.env` file so they can be accessed anywhere via `process.env`.
- **ChatModule**: Imports our custom Chat feature into the main app.

## 3. Data Transfer Objects (`src/chat/dto/chat-request.dto.ts`)
DTOs (Data Transfer Objects) define the shape of the data we expect to receive from the client.
- **`@IsString()` & `@IsNotEmpty()`**: Decorators that ensure the client cannot send an empty request or a number instead of text. If they do, the API will automatically reject it with a `400 Bad Request`.
- **`@ApiProperty()`**: Tells Swagger what this property is so it can document it properly.

## 4. The Controller (`src/chat/chat.controller.ts`)
Controllers are responsible for handling incoming HTTP requests and returning responses to the client.
- **`@Controller('chat')`**: Defines the base route for this controller (`http://localhost:3100/chat`).
- **`@Post()`**: This endpoint only listens to POST requests.
- **`processChat` method**: It receives the validated DTO from the client, hands the prompt over to the `ChatService`, and then returns the AI's answer back to the client.

## 5. The Service (`src/chat/chat.service.ts`)
Services contain all the heavy business logic. This is the brain of the operation.
- **Initialization**: When the service starts, it creates a new instance of the OpenAI client using your `OPENAI_API_KEY`.
- **`processEnquiry`**: 
  1. It builds an array of messages, starting with a System prompt (the AI's personality and instructions) and the User prompt.
  2. It defines the `tools` array. This tells the AI what external functions it has access to (searching laptops, converting currencies).
  3. It sends the first request to OpenAI (`gpt-3.5-turbo`).
  4. If the AI decides it needs to use a tool (e.g., the user asked about a product), it intercepts that request via `tool_calls`.
  5. It executes our local TypeScript methods (`searchProducts` or `convertCurrencies`) based on what the AI asked for.
  6. It injects the result of those methods back into the conversation history and makes a *second* call to OpenAI.
  7. Finally, it returns the natural language response generated by OpenAI.

### Tools Explained:
- **`searchProducts`**: Reads the `products_list.csv` using a file stream. It compares the user's query against every column. To save memory, as soon as it finds exactly 2 matches, it destroys the stream and returns those two items.
- **`convertCurrencies`**: Makes an HTTP request to `openexchangerates.org`. Since the free tier uses USD as the base currency, it performs "triangulation" (converting the starting currency to USD, and then from USD to the target currency) to return an accurate exchange rate.
